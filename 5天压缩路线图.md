## Day 1：混合检索（BM25 + Dense 双路融合）

### 目标
把当前纯向量搜索升级为 **Dense（语义） + Sparse（BM25）** 双路并行检索，RRF 融合排序。

### 新增文件

| 文件 | 说明 |
|------|------|
| `Vector_Store/hybrid_retriever.py` | 混合检索器：两路并行召回 + RRF 融合 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `Vector_Store/base_vector_store.py` | 新增 `_bm25_encode()` 工具方法 |
| `main.py` | 新增 `/hybrid_search` 命令，调用混合检索 |

### 新增类

```
HybridRetriever
├── __init__(dimension, bm25_encoder)
├── build_index(texts, dense_vectors)    → 建立 dense 索引 + BM25 索引
├── search(query, query_vec, top_k)      → 双路并行 + RRF 融合
└── add_texts(texts, dense_vectors)      → 增量添加
```

### 关键技术点

```
两路并行:
  路 A: Dense → IndexFlatIP 搜索 → top_k×5 条
  路 B: Sparse → BM25 相似度 → top_k×5 条

RRF 融合:
  score = 1/(k + rank_dense) + 1/(k + rank_sparse)  其中 k=60
```

### 验收标准

```
输入:  /hybrid_search 机器学习 5
输出:  5 条结果，每条带 hybrid_score
预期:  混合检索结果质量 > 纯向量检索
```

## Day 2：LLM 智能检索（查询改写 + Self-Querying + 自动打标）

### 目标
接入 LLM API，实现文档自动分类/摘要/关键词提取，以及智能查询改写。

### 新增文件

| 文件 | 说明 |
|------|------|
| `llm_client.py` | LLM API 封装（支持通义千问/DeepSeek/GLM） |
| `query_rewriter.py` | 查询改写（指代消解 + 同义词扩展） |
| `self_querying.py` | 意图识别（从自然语言提取过滤条件） |
| `auto_tagger.py` | 文档打标（分类 + 摘要 + 关键词） |

### 新增类

```
LLMClient
├── rewrite(query, history)         → 改写后的 query
├── self_query(query, schema)       → {semantic_query, filters}
├── tag_document(text, schema)      → {category, summary, keywords}
└── _call(messages)                 → LLM API 调用 + 重试
```

### 关键技术点

```python
# Self-Querying Prompt 核心
"""
用户输入: "{query}"
可用分类: {schema_enum_values}
输出 JSON: {"semantic_query": "...", "filters": {...}}
规则: 明确提到分类词才加 filter，产品名称/型号走语义检索不做filter
"""
```

### 验收标准

```
输入:  /search 制造业的成功案例
输出:  self_query → {semantic_query: "成功案例", filters: {industry: "制造业"}}
```

## Day 3：入库流水线（清洗 + 两阶段切分 + 质量评分 + 数据库持久化）

### 目标
从 JSON 文件升级为 SQLite 持久化，实现完整入库流水线。

### 新增文件

| 文件 | 说明 |
|------|------|
| `db_models.py` | SQLAlchemy 数据模型（document / chunk / kb / log） |
| `text_cleaner.py` | 4 阶段文本清洗流水线 |
| `quality_scorer.py` | 4 维度质量评分 |
| `chunk_pipeline.py` | 两阶段切分（Segment → Chunk + overlap） |
| `ingestion_pipeline.py` | 入库流水线编排器 |

### 新增类

```
TextCleaner
├── clean_stage1(text)    → 编码清洗
├── clean_stage2(text)    → 格式噪声
├── clean_stage3(text)    → 内容噪声
└── clean(text)           → 完整 4 阶段 + 双层输出

QualityScorer
├── completeness(pages)   → 完整度（30%）
├── structure(text)       → 结构度（30%）
├── density(text)         → 内容密度（20%）
└── clean_score(signals)  → 清洗分（20%）

TwoLevelChunker
├── segment(text)         → 章节级（3000~6000 字符）
└── chunk(segment)        → 句子级（400 字符，overlap 2 句）
```

### 验收标准

```
/python -c "from chunk_pipeline import TwoLevelChunker; c=TwoLevelChunker(); print(len(c.chunk(c.segment('测试文本')[0])))"
→ 正常切分不报错
```

## Day 4：FastAPI 接口 + 端到端链路

### 目标
从 CLI 变成 Web 服务，所有能力通过 HTTP 接口暴露。

### 新增文件

| 文件 | 说明 |
|------|------|
| `api.py` | FastAPI 应用主文件 |
| `schemas.py` | Pydantic 请求/响应模型 |

### API 接口设计

| 方法 | 路径 | 功能 |
|:----:|------|------|
| POST | `/search` | 智能检索（自动 query rewrite + self-querying + hybrid search） |
| POST | `/upload` | 上传文档，触发异步入库 |
| GET  | `/documents` | 文档列表 |
| GET  | `/documents/{id}` | 文档详情 + 状态 |
| GET  | `/health` | 健康检查 |

### 关键技术点

```python
@app.post("/search")
async def search(request: SearchRequest):
    # 1. Query rewriting
    query = await llm.rewrite(request.query, request.history)
    # 2. Self-Querying
    sq = await llm.self_query(query, schema)
    # 3. Hybrid search
    results = retriever.search(sq.semantic_query, ...)
    # 4. 3D weighting
    scored = three_dimension_weight(results)
    return {"results": scored}
```

### 验收标准

```
curl -X POST "http://localhost:8000/search" -H "Content-Type: application/json" -d '{"query":"机器学习"}'
→ 返回 JSON 格式的检索结果
```
