## 一、项目概述

这是一个**从零手写的文本向量化 + 向量检索学习项目**，共分 9 个阶段逐步搭建。系统将中文/英文文本通过 Embedding 模型转为向量，存储在向量数据库中，支持相似度搜索，并提供了交互式 CLI 界面。

## 三、各模块详解

### 1. `embedding.py` — 文本向量化服务

**作用**：将文本转为数值向量，是整个系统的数据入口。

**类**：`EmbeddingService`

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `__init__` | model_name, device | 无 | 加载模型，获取向量维度 |
| `encode(text)` | `str`（单条文本） | `list[float]`（向量） | 单条文本转向量 |
| `encode_batch(texts)` | `list[str]`（文本列表） | `list[list[float]]`（向量列表） | 批量文本转向量 |
| `dimension` (property) | 无 | `int` | 返回向量维度 |

**技术要点**：
- 使用 `fastembed` 库，默认模型：`BAAI/bge-small-zh-v1.5`
- 模型输出向量维度：**512 维**（最初误以为是 384 维）
- `_init_model_` 用局部变量 `model` 而非直接赋值 `self.model`，待函数返回后再赋值

---

### 2. `vector_store_faiss.py` — FAISS 暴力搜索索引

**作用**：基于 FAISS 的 `IndexFlatIP` 实现暴力搜索（扫描全部向量），当前系统的**默认索引**。

**类**：`FaissVectorStore`

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `__init__(dimension)` | 向量维度 | 无 | 创建 IndexFlatIP |
| `add(text, vector)` | 文本 + 向量 | 无 | 单条添加 |
| `add_batch(texts, vectors)` | 文本列表 + 向量列表 | 无 | 批量添加 |
| `add_from_file(path, emb, method)` | 文件路径 + embedding 实例 + 分块方式 | 无 | 从文件加载 |
| `search(query_vec, top_k)` | 查询向量 + K 值 | `list[dict]` | 搜索相似文本 |
| `save(path)` | 路径前缀 | 无 | 保存索引 + 文本 |
| `load(path)` | 路径前缀 | 无 | 加载索引 + 文本 |
| `count` (property) | 无 | `int` | 返回向量总数 |

**技术要点**：
- 使用 `IndexFlatIP` + `normalize_L2`，使内积等价于余弦相似度
- save 生成两个文件：`{path}.faiss`（FAISS 索引）+ `{path}_texts.json`（文本列表）
- **注意**：`search` 方法接收的是**向量**而非文本，调用方需先 `emb.encode(query)`

---

### 3. `vector_store_ivf.py` — IVF 倒排索引

**作用**：使用 `IndexIVFFlat`，通过 K-means 聚类将向量空间分区，搜索时只检索最近的几个分区，提升搜索速度。

**类**：`IvfVectorStore`

**关键参数**：
- `nlist=100`：聚类中心数（越大搜索越快但精度下降）
- `nprobe=10`：搜索时检查的分区数（越大越准但越慢）

**特殊逻辑**：
- 首次添加数据时才训练索引（`is_trained` 标志）
- 小数据集时动态调整 `nlist = min(self.nlist, n_train)`，避免训练失败
- save/load 时会保存/恢复 `is_trained` 状态

**与 flat 对比**：

| 对比项 | Flat | IVF |
|--------|------|-----|
| 搜索方式 | 扫描全部 | 只搜最近分区 |
| 速度 | 慢（数据量大时） | 快 |
| 精度 | 100% | 近似（略低于 flat） |
| 训练 | 不需要 | 需要先训练 |

---

### 4. `vector_store_hnsw.py` — HNSW 分层图索引

**作用**：使用 `IndexHNSWFlat`，通过多层导航图实现超快速近似搜索，速度通常比 IVF 更快。

**类**：`HnswVectorStore`

**关键参数**：
- `M=32`：每个节点的最大连接数（越大越准但越耗内存）
- `efConstruction=128`：建图时的搜索范围
- `efSearch=64`：搜索时的候选集大小

**三种索引速度对比（理论）**：

```
Flat:  100% 精确  ·  速度慢  ·  不训练
IVF:   近似精确  ·  速度快  ·  需训练
HNSW:  近似精确  ·  最快    ·  需建图
```

---

### 5. `simple_vector_store.py` — 手写向量库（已弃用）

**作用**：纯 Python 实现，不依赖 FAISS。使用 Python 列表存储向量，自己实现余弦相似度计算和排序。

**保留意义**：作为理解向量检索原理的参考代码，帮助理解 FAISS 底层做了什么。

---

### 6. `document_loader.py` — 文档加载与分块

**作用**：读取 `.txt` 文件，并提供多种文本分块策略。

| 函数 | 策略 | 说明 |
|------|------|------|
| `read_file(path)` | — | 读取 txt 文件，返回字符串 |
| `chunk_by_sentence(text)` | 按句 | 用正则 `[。！？]` 分割 |
| `chunk_by_paragraph(text)` | 按段 | 用 `\n\n` 分割 |
| `chunk_by_size(text, size, overlap)` | 按固定字数 | 滑窗截取，可重叠 |
| `chunk_by_jieba(text, max_words)` | 按分词 | 先用 jieba 分词，再按词数分组 |
| `filter_stopwords(words)` | 去停用词 | 过滤"的、了、是"等常见词 |

---

### 7. `main.py` — CLI 交互入口

**作用**：提供命令行界面，串联所有模块。

**支持命令**：

| 命令 | 用法 | 说明 |
|------|------|------|
| `/help` | `/help` | 显示帮助 |
| `/search <query>` | `/search 机器学习` | 搜索（显示耗时 ms） |
| `/search_jieba <query>` | `/search_jieba 机器学习` | 先分词再搜索 |
| `/add <file> [method]` | `/add sample.txt sentence` | 从文件添加 |
| `/count` | `/count` | 查看当前总数 |
| `/switch <type>` | `/switch ivf` | 切换索引：flat/ivf/hnsw |
| `/save <path>` | `/save data/my_db` | 保存到磁盘 |
| `/load <path>` | `/load data/my_db` | 从磁盘加载 |
| `/exit` | `/exit` | 退出 |

**注意**：
- 切换索引后**数据清空**，需要重新添加
- `/switch` 切换时旧索引数据仍在内存中但失去引用（Python GC 会回收）
- 不支持修改 `top_k`，固定返回 5 条

## 四、数据流总图

```
用户输入 "机器学习"
       │
       ▼
emb.encode("机器学习")       ← 文本 → 向量
       │
       ▼
db.search(vec, top_k=5)     ← FAISS 搜索
       │
       ▼
返回 [{"text": "...", "score": 0.92, "index": 0}, ...]
       │
       ▼
main.py 格式化打印到屏幕
```

**完整文件→搜索链路**：
```
sample.txt
    → read_file() → 字符串
    → chunk_by_sentence() → ["句子1", "句子2", ...]
    → emb.encode_batch(chunks) → [[vec1], [vec2], ...]
    → db.add_batch(chunks, vectors) → 存入 FAISS 索引
    → 用户搜索 → emb.encode(query) → db.search(vec) → 返回结果
```

## 六、已知问题与改进空间

### 已知 Bug
1. **`FaissVectorStore.search` 接收文本名但实际需传向量** — 方法签名写的 `text: str`，但实现中直接 `np.array([text])` 会导致运行时出错。目前靠 main.py 调用前先 `encode` 来绕过，但如果别人独立调用这个类就会出 bug。

2. **`FaissVectorStore.add_from_file` 不支持 `jieba` 分块** — 只有 ivf 和 hnsw 版本支持，flat 版缺少 `chunk_by_jieba` 分支。

### 可改进点
1. **统一所有 vector store 的 `search` 接口** — 让每个类都在内部存一份 `embedding_service`，外部调用 `db.search("文本")` 即可，无需手动 encode
2. **`/switch` 保留数据** — 切换索引时重建索引而非清空数据
3. **支持自定义 `top_k`** — 固定 5 条不够灵活
4. **批量添加测试数据** — 用脚本自动生成 100+ 条数据，测试三种索引的真实性能差异
5. **异常处理** — 文件不存在、格式错误、搜索空库等场景缺少 try/except

## 十、运行环境（回自己电脑配置用）

```
Python 3.11.9
```

本项目用的是 `.venv` 虚拟环境（在项目根目录下）。

**激活方式**：

```bash
# Windows
.venv\Scripts\activate

# Mac / Linux
source .venv/bin/activate
```

激活后运行：

```bash
python main.py
```

### 核心依赖包

| 包名 | 版本 | 用途 | 安装命令 |
|------|------|------|----------|
| **fastembed** | 0.8.0 | 文本转向量（核心） | `pip install fastembed` |

在新电脑上（已装好 Python 3.11），在项目目录下执行：

```bash

python -m venv .venv

# 2. 激活
.venv\Scripts\activate      # Windows
# source .venv/bin/activate  # Mac/Linux

# 3. 安装依赖
pip install fastembed faiss-cpu numpy jieba
```

> `fastembed` 安装时会自动拉 `onnxruntime`、`huggingface_hub`、`tokenizers` 等依赖，不需要手动装。

